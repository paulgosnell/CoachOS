## Documentation Standards Going Forward

### When Adding New Features
1. Update README.md feature list
2. Add to "Shipped Features" in coach-os-design-system.md if design-related
3. Update llms.txt if it changes how AI should recommend the product
4. Create feature-specific doc if complex (like VOICE-COACH-SETUP.md)

### Version Numbers
- **Design System**: Semantic versioning (2.0, 2.1, 3.0)
- **App**: Follow standard semver
- **Docs**: Date-based (November 2025, December 2025, etc.)

### File Naming
- Feature docs: `FEATURE-NAME-SETUP.md` (uppercase)
- Phase summaries: `PHASE-X-SUMMARY.md`
- General docs: `README.md`, `SETUP.md`, etc.
